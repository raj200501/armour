# Datasets

Datasets here are generated from repo scripts so benchmark outputs are
reproducible.

Current dataset:

- `local_agent_labeled_traces.jsonl`
- `reviewed_external_agent_traces.jsonl`
- `live_external_agent_runs.jsonl`
- `adjudication_queue_v3.jsonl`
- `model_agent_run_plan.json`

Generate it:

```bash
python3 scripts/generate_labeled_dataset.py
```

Each record contains:

- eval id
- local agent id
- policy name
- reference label
- label source
- MCP-style event log

The current labels are synthetic reference labels. They are useful for
integration metrics and regression tests, but they should be replaced or
augmented with independent human labels before making external performance
claims.

`reviewed_external_agent_traces.jsonl` contains reviewed external-style MCP
trace fixtures with two consensus review records per item. It includes known
false-positive and false-negative cases for monitor error analysis.

`live_external_agent_runs.jsonl` is generated by executing
`examples/external_agents/tiny_mcp_agent.py` through
`scripts/run_live_external_agent_benchmark.py`. It is a live external-harness
trace dataset, not a frontier-model dataset.

`adjudication_queue_v3.jsonl` contains records where Armour, the offline judge,
or the reference labels disagree.

`model_agent_run_plan.json` is a dry-run manifest for Benchmark v4. It records
which model-backed agent traces would be collected without making network calls
or storing credentials. A real local run writes `model_agent_runs.jsonl`; those
records should be independently reviewed before external claims.

Reviewer packet artifacts are generated under `review/`, not `datasets/`,
because they are workflow materials rather than benchmark datasets. The blind
queue hides reference labels; the internal queue keeps them for adjudication.
